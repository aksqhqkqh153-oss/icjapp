이청잘 앱 프로젝트 운영환경 메모
작성일: 2026-03-25

1. 사용 중인 서버 / 배포 연동
- Cloudflare 사용 중
  - 용도: 프론트엔드 배포(Cloudflare Pages), 도메인/DNS/CDN/캐시
- Railway 사용 중
  - 용도: 백엔드 API(FastAPI) 배포, 헬스체크(/api/health), 운영 런타임

2. 사용 중인 DB (코드 분석 결과)
- 운영 기준 우선 DB: Railway PostgreSQL
  - 근거:
    - backend/app/db.py 에서 DATABASE_URL 이 postgres:// 또는 postgresql:// 로 시작하면 DB_ENGINE 을 postgresql 로 설정함.
    - docs/08_Railway_PostgreSQL_전환_빠른절차.md, railway 관련 문서들에서 DATABASE_URL=${{Postgres.DATABASE_URL}} 사용을 전제로 안내함.
- 로컬/폴백 DB: SQLite
  - 파일 위치: backend/data/app.db
  - 근거:
    - backend/app/db.py 에 SQLITE_DB_PATH = backend/data/app.db 로 정의됨.
    - DATABASE_URL 이 없으면 sqlite 로 동작하도록 되어 있음.
- 정리:
  - 운영 배포: Railway PostgreSQL 사용 전제
  - 로컬 개발/임시 폴백: SQLite(app.db)

3. 사용 중인 저장소
- Git 사용 중
- GitHub 저장소 연동 후 수시 최신화/재배포 중

4. 현재 운영 방식
- GitHub 에 push 하면 Cloudflare Pages 와 Railway 가 각각 최신 커밋 기준으로 재배포될 수 있음.
- Cloudflare 와 Railway 는 순차 연동이 아니라 병렬/개별 배포 대상으로 봐야 함.
- 프론트 수정 반영 시 아래 2가지를 함께 확인해야 함.
  1) Cloudflare Pages: frontend 기준 빌드 반영 여부
  2) Railway: backend/static 기준 프론트 빌드본 반영 여부

5. 프론트엔드 구조 관련 중요 메모
- 이 프로젝트는 프론트가 이중 관리 구조임.
  - 소스 프론트: frontend/src/*
  - Railway 에서 실제 서빙될 수 있는 정적 빌드본: backend/static/*
- 따라서 화면 구조 변경 시 단순히 frontend/src 만 바꾸면 안 되고,
  - 빌드 결과가 backend/static/index.html 및 backend/static/assets/* 에도 반영되었는지 확인 필요
- 사용자가 실제 보는 주소가 Cloudflare Pages 인지, Railway/백엔드 정적 서빙인지 항상 먼저 확인할 것

6. 배포 시 주의사항
- Git 저장소 폴더(.git 포함)에서만 git add / commit / push 수행할 것
- ZIP 압축 해제 폴더는 그대로 Git 저장소가 아닐 수 있으므로, 기존 clone 폴더에 덮어쓰거나 새로 clone 후 반영할 것
- Cloudflare Pages 는 네트워크 타임아웃(npm clean-install ETIMEDOUT) 가능성이 있으므로 재시도 로그 확인 필요
- Railway 는 /api/health 통과 여부 확인 필요

7. 데이터 관련 주의사항
- 운영 DB 가 실제로 Railway PostgreSQL 에 연결되어 있으면 일반 재배포만으로 데이터는 보통 유지됨
- 단, 초기화 코드/seed 덮어쓰기/파괴적 migration 이 있으면 데이터 변경 가능성이 있음
- 로컬 SQLite(app.db)는 운영 영속 DB로 간주하지 말 것

8. 이후 GPT 에 프로젝트 업데이트 요청 시 반영할 원칙
- Cloudflare + Railway + Git 운영 구조를 전제로 수정
- 운영 DB 는 Railway PostgreSQL 기준으로 가정하되, 로컬 개발은 SQLite 폴백도 함께 고려
- 프론트 수정 시 frontend/src 와 backend/static 반영 관계를 반드시 함께 점검
- 배포 오류가 있으면 Cloudflare Pages 로그와 Railway 로그를 함께 보고 판단
- 가능한 경우 빌드/문법 점검 후 ZIP 완성본으로 제공

9. 확인 권장 항목
- Cloudflare Pages 설정:
  - Root directory: frontend
  - Build command: npm run build
  - Output directory: dist
- Railway 환경변수:
  - DATABASE_URL
  - PORT
  - 기타 백엔드 운영 변수
