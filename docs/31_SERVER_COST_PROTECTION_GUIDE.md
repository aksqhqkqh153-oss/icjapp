# 서버 비용 보호 / 봇·크롤링 방지 가이드

## 이번 업데이트에 포함된 내용
- FastAPI 전역 HTTP 미들웨어 기반 IP 요청 제한
- 인증 경로(`/api/auth/*`) 별도 강한 제한
- 공개 페이지(`/p/*`, `/public/p/*`, `/share/*`) 별도 조회 제한
- 공개 읽기 API(`/api/public/*`, `/api/questions*`) 별도 조회 제한
- 대표적인 자동화 User-Agent 차단
- `robots.txt` 에서 민감 API 수집 억제
- 관리자 화면에서 현재 보호 정책과 예시 설명 확인 가능

## 왜 필요한가
1. 로그인·회원가입·SMS 인증 폭주를 막아 외부 연동 비용을 줄임
2. 공개 프로필 대량 수집을 줄여 DB/대역폭 비용을 낮춤
3. 단순 스크립트형 크롤러를 초기에 차단해 운영 안정성을 높임

## 현재 기본값
- 전체 IP: 60초당 240회
- 인증 경로: 600초당 30회
- 공개 페이지: 60초당 90회
- 공개 읽기 API: 60초당 120회

## 추가 권장 사항
- Cloudflare WAF Rate Limiting도 함께 적용
- 이미지/공개 프로필 HTML은 CDN 캐시 사용
- 비회원 쓰기 기능은 Turnstile 필수 적용 유지
- 429/403 로그 급증 시 알림 연동
- 업로드 크기와 월간 미디어 사용량을 주기적으로 점검

## 주의
현재 구현은 애플리케이션 인스턴스 메모리 기반 제한입니다.
단일 인스턴스/소규모 서비스에는 효과적이지만, 다중 인스턴스 환경에서는 Cloudflare/WAF 또는 Redis 기반 중앙 제한과 병행하는 것이 더 안정적입니다.
