[4. 배포 추천 (사용자 30~50명 / 이미지 다수 첨부)]
- 사용량 가정: 일정 1건당 이미지 30MB, 하루 10건 정도면 하루 약 300MB, 한 달 약 9GB의 신규 업로드 데이터 발생 가능
- 무료 최선(테스트용): Render Free + Cloudflare R2
  * Render Free는 무료로 Python/FastAPI 웹 서비스를 배포할 수 있지만, 15분 무접속 시 서비스가 잠들고, 로컬 파일이 유지되지 않으므로 실운영용으로는 부적합
  * 이미지는 Cloudflare R2에 저장 권장
- 저비용 차선(실운영 권장): Railway Hobby + Cloudflare R2 + Cloudflare DNS
  * Railway Hobby는 월 $5 최소 사용 구조이며, 포함 크레딧 범위 안에서는 소규모 운영에 적합
  * Cloudflare R2는 10GB 무료, 이후 표준 스토리지 $0.015/GB-month, egress 요금 0원 구조라 이미지 조회가 많은 앱에 유리
  * 장기 보관이 많고 다운로드 비중이 낮으면 Backblaze B2도 후보이나, 모바일 앱 이미지 조회 중심이면 R2가 운영이 단순함
- 추가 권장:
  * 현재 DB는 SQLite이므로, 실운영 전에는 PostgreSQL 전환을 검토할 것
  * 이미지와 첨부파일은 서버 로컬 디스크가 아니라 오브젝트 스토리지(R2/B2)에 저장할 것

[6. 안드로이드 패키징에 필요한 프로그램]
- Windows PC
- Node.js / npm
- Android Studio
- Android SDK
- Android 15(API 35) SDK
- Java (Android Studio 번들 JDK 또는 호환 JDK)

[7. 안드로이드 패키징 진행 순서]
1) mobile_capacitor 폴더로 이동
2) npm install 실행
3) npm run init:android 실행
4) npm run sync:android 실행
5) npm run open:android 실행
6) Android Studio에서 패키지명, 앱 아이콘, 스플래시, 버전명, versionCode, targetSdkVersion 확인
7) 서명키 생성 후 AAB 빌드
